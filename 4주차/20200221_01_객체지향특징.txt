
 + 캡슐화, Encapsulation
	 하나의 객체를 표현할 때 멤버 필드(정보), 멤버 메소드(기능)을
	하나의 클래스의 멤버로 모아놓는 것

	외부의 접근 제한하면서 데이터를 보호한다. (정보은닉)


 + 상속, Inheritance
	작성되어있는 클래스를 이용하여 새로운 클래스를 작성할 때 적용

	이미 만들어져있던 코드를 새로 작성하지 않고 상속받아 사용하는 것

	재사용성이 높아진다
	코드 중복을 줄여준다


 + 다형성, Polymorphism
	부모클래스 타입으로 다양한 자식클래스 타입의 객체를 다룰 수 있다

	클래스 별로 작성될 메소드를 하나의 이름으로 만들 수 있다
